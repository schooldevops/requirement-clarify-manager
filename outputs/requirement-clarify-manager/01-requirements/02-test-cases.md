# 테스트 케이스 (리스크 기반 통합 시나리오)

총 36개의 포괄적인 TC(정상/예외/경계/보안)를 정의했습니다.

| TC-ID | 분류 | 관련 요건/EC | 검증 시나리오 | 테스트 입력값/액션 | 기대 결과 | 중요도 |
|---|---|---|---|---|---|---|
| **[요구사항 입력 - 수기 및 일반 검증]** | | | | | | |
| TC-001 | 수기입력-정상 | FR-001 | 요구사항 직접 입력 정상 저장 | 모든 필수 정보(프로젝트명, 요구사항번호 등) 기입 후 '저장' | DB에 정상 등록되며 상태가 'Draft'로 표시됨 | Critical |
| TC-002 | 수기입력-예외 | EC-002 | 필수 필드 누락 방어(Jira ID) | '요구사항 번호' 필드를 비워두고 저장 시도 | ERR-VAL-001 로깅 및 화면 하이라이트, 저장 실패 | High |
| TC-003 | 수기입력-예외 | EC-002 | 필수 필드 누락 방어(요구사항명) | '요구사항 이름' 필드를 비워두고 저장 시도 | ERR-VAL-001 발생, 저장 차단 | High |
| TC-004 | 수기입력-예외 | EC-007 | 비정상 버전 포맷 검증 | 버전에 `1.0`, `v1. final` 등을 입력 | 정규식(M.m.p) 위반으로 폼 검증 에러 발생 | Medium |
| TC-005 | 수기입력-정상 | FR-001 | 정상 버전 포맷 입력 | 버전에 `1.0.0`, `2.1.13` 입력 | 정상적으로 파싱 후 저장됨 | Medium |
| TC-006 | 수기입력-경계 | EC-008 | 비존재 연관 요구사항 매핑 | 존재하지 않는 DB 상의 JIRA ID를 연관 번호에 입력 | 경고 모달 노출(무시하고 저장 가능, 또는 정책상 차단) | Medium |
| TC-007 | 수기입력-예외 | EC-003 | JIRA ID(요구사항번호) 중복 | 이미 등록된 JIRA ID(예: `REQ-100`)로 신규 등록 시도 | ERR-DUP-001 반환 및 리비전업(버전업) 가이드 제공 | High |
| **[요구사항 입력 - 파일 업로드]** | | | | | | |
| TC-008 | 파일입력-정상 | FR-001 | 정상 구조 .md 파일 파싱 | 표준 포맷의 .md 파일 업로드 | 본문 및 메타 필드가 폼에 자동 바인딩됨 | Critical |
| TC-009 | 파일입력-경계 | EC-001 | 파일 사이즈 10MB 초과 제한 | 12MB 크기의 텍스트가 담긴 .md 파일 업로드 시도 | 브라우저/서버 단에서 사이즈 체크로 업로드 즉시 차단 (ERR-FILE-001) | High |
| TC-010 | 파일입력-경계 | FR-001 | 대용량 텍스트 안정성 | 본문 필드에 4~5MB 길이 텍스트 복사/붙여넣기 후 등록 | 페이로드 잘림 현상 없이 DB에 CLOB/TEXT로 온전하게 저장 및 조회됨 | High |
| TC-011 | 파일입력-예외 | EC-010 | 미허용 파일 확장자 차단 | `.doc`, `.pdf`, `.txt` 파일 업로드 시도 | 확장자 미스매치로 파일 선택 거절 혹은 ERR-FILE-002 발생 | Medium |

| TC-014 | 상태전이-정상 | FR-002 | Clarifying -> Clarified 전이 | 명확화 질의/응답 100% 완료 후 상태 변경 | 분석 완료 상태로 정상 세팅됨 | Critical |
| TC-015 | 상태전이-정상 | FR-002 | Clarified -> In Progress 전이 | 개발 진행 탭으로 넘기는 액션 | 외부 시스템 연동(선택) 및 상태 업데이트 성공 | High |
| TC-016 | 상태전이-정상 | FR-002 | In Progress -> Done 전이 | 작업 완료 처리 | 진행 상태 변경 및 수정 불가(Read Only) 락 적용 | High |
| TC-017 | 상태전이-예외 | EC-004 | 비정상 상태 건너뛰기 1 | Draft에서 임의 API 호출로 즉시 Done으로 변경 시도 | 서버단 워크플로우 제어 로직에 의해 차단 및 ERR-STS-001 발생 | High |
| TC-018 | 상태전이-예외 | EC-004 | 비정상 상태 건너뛰기 2 | Clarifying 중단하고 바로 In Progress로 변경 시도 | 필수 전제(명확화 완료) 미달로 차단 | High |
| TC-019 | 상태전이-예외 | FR-002 | 완료건 불법 수정 방어 | Done 상태인 요구사항 문서의 수정(PUT) API 호출 | HTTP 403 인증/인가 에러 혹은 "완료건 수정 불가" 에러 발생 | Critical |
