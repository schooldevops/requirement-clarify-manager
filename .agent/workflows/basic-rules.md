# Basic Rules

- 모든 요구사항은 Basic Rules 를 항상 준수 해야한다. 
- 모든 산출물은 한국어로 작성해야한다.
- 모든 산출물은 Git Commit Message 형식에 맞게 작성해야한다.
- 모든 요구사항을 LLM이 작업할 때에는 반드시 다음 형식으로 outputs/added-requirements/<요구사항명>-Plan-<날짜/시간>.md 파일을 작성하고, 다음에 기술하는 파일 형식에 맞게 작업하고, 작업 파일을 참조하여 작업을 진행해야한다.

## 플랜 형식

- 플랜 파일은 outputs/added-requirements/<요구사항명>-Plan-<날짜/시간>.md 형식으로 작성한다.
- 플랜 파일은 반드시 한국어로 작성한다.
- 플랜 파일은 반드시 Git Commit Message 형식에 맞게 작성한다.
- 플랜 파일은 반드시 다음에 기술하는 파일 형식에 맞게 작성한다.

- 파일 내용

# [요구사항명]

## 1. 요구사항 내용

## 2. 기존 소스 형식 분석

- 만약 기존 소스 구조를 이미 알고 있다면, 기존 소스 구조해당 내용을 활용한다. 
- 소스의 구조
- 소스에서 변경해야할 영역 확인 
    - 변경 영역은 반드시 국소적으로 최소화 해야한다. 

## 3. 작업계획 

- 작업계획은 [x] 형식으로 실제 작업할때 단계적으로 수행할 수 있도록 해야한다. 
- 매 작업 계획 단계는, 나에게 작업 계획의 실행 여부를 물어보고, 진행해야한다. 
- 작업 계획에서 승인이 나면, 해당 작업계획을 차례대로 실행한다. 

## 4. 작업 결과

- 실제 수정된 소스와 수정 내역을 요약하여 작성한다.
